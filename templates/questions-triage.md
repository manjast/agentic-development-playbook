# Question Triage

Purpose: keep unresolved questions separate from the task backlog and decision log.

Use this file when a project has enough open questions that they no longer fit cleanly inside task cards, meeting notes, or the PoC brief.

## How to use it

- Capture each meaningful unresolved question here.
- For each question, record either repo evidence or a named owner / next step.
- When a question becomes actionable work, link or create a task in `TASKS.md`.
- When a question is resolved, keep a short answer snapshot and the evidence link.

This file is for unresolved questions. It is not the ordered execution backlog (`TASKS.md`) and it is not the committed decision log (`DECISIONS.md`).

## Suggested status values

- `open` — tracked and still unresolved
- `answered` — resolved with enough evidence to close
- `partial` — partly answered, but more evidence or scope is needed
- `conflict` — evidence or stakeholder answers disagree
- `needs_owner` — unresolved and no clear owner / next step yet
- `deferred` — intentionally postponed
